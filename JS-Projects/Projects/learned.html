What I've Learned about Learning to Code

  1. Be Patient. There is a lot to learn. It's going to take time.
  2. Every morning, make a list of what you want to accomplish that day.
  3. Approach everything with the confidence that I will soon be able to teach this to others.
  4. When I get stuck, take a break and come back.
  5. If I can't figure something out, go through this list in order:
    • DevTools
    • Console.log
    • ChatGPT
    • Source Websites
    • Google Searching
    • YouTube Searching
    • Post a Question on Message Boards
  6. Maintain a balance of Training Materials, YouTube Tutorials and Hands-on Practice.
  7. The first step in creating a successful project is breaking it down into smaller parts. And maybe breaking those parts down even more.
  8. When something doesn't work, sometimes it's because I'm skipping a step. Read through the code again and make sure I'm not trying to do two things at once.
  9. Take time to enjoy what you create, and also see if you can 'break' it from a user perspective.
